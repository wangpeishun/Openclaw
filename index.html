🚀 部署方式

多种部署方案，按需选择

从云端一键部署到本地源码安装，满足不同用户的技术门槛和使用场景。

推荐
☁️

阿里云轻量应用服务器

官方适配镜像，配置推荐 2vCPU + 2GB 内存（最低 2GB），一键部署，5 分钟上线。

  1. 访问阿里云轻量服务器控制台，镜像选择「OpenClaw」
  2. 配置推荐：2vCPU + 2GB 内存，地域选「美国弗吉尼亚」
  3. 登录百炼控制台，在「密钥管理」创建并复制 API-Key
  4. 进入实例「应用详情」，一键放通 18789 端口
  5. 粘贴 API-Key，执行命令获取专属 Token 即可访问
官方镜像 最低 ¥60/月 5 分钟部署
🌩️

金山云 / 腾讯云

国内云厂商均已支持一键部署，可直接对接飞书、钉钉等国内平台。

  1. 在对应云市场搜索「OpenClaw」应用镜像
  2. 购买轻量云服务器，选择应用镜像
  3. 配置大模型 API-Key（千问 / GLM / MiniMax 均可）
  4. 对接飞书/钉钉 Bot 完成 Gateway 配置
国内平台 飞书/钉钉集成
💻

联想想帮帮（线上/线下）

联想官方知识库提供一键部署服务，内置 51 个 Skills，无需任何技术背景。

  1. 访问联想官网知识库首页，进入 OpenClaw 部署入口
  2. 下载安装「想帮帮」客户端
  3. 点击左侧 OpenClaw 入口，选择「立即部署」
  4. 系统全自动完成下载、安装、配置流程
零代码 51 个内置技能 线下到店服务
01

环境准备

确保已安装 Node.js（≥ v18），可通过 node -v 验证版本。

Terminal 
node -v    # 需要 v18+
npm -v
02

全局安装 OpenClaw

使用 npm 全局安装，安装后可在任意目录使用 openclaw 命令。

Terminal 
npm install -g openclaw@latest
03

执行初始化引导

运行 onboard 命令完成交互式初始化，自动生成 ~/.openclaw 配置目录。

Terminal 
openclaw onboard
04

配置大模型接入

在 ~/.openclaw/config.yaml 中配置模型 API-Key。支持 Claude、GPT-5.4、Gemini 3.1、DeepSeek、千问等。

~/.openclaw/config.yaml 
model:
  primary: claude-opus-4
  fallback: gpt-5.4
  api_key: sk-your-key-here
gateway:
  telegram:
    token: your-telegram-bot-token
05

启动服务

Terminal 
openclaw start
01

克隆仓库

Terminal
git clone https://github.com/OpenClaw/openclaw.git
cd openclaw
02

安装依赖并构建

Terminal
npm install
npm run build
03

配置 SSL（可选，推荐）

使用 Let's Encrypt 申请证书，配合 Nginx 反向代理开启 HTTPS 访问。源码部署建议配合 Nginx 使用。

Terminal
# 安装 certbot
apt install certbot python3-certbot-nginx
# 申请证书
certbot --nginx -d yourdomain.com
⚠️
注意：OpenClaw 更新非常频繁，约半个月一个版本。 生产环境使用建议追踪官方 Release，升级前务必备份配置文件和记忆数据。
01

拉取官方镜像并运行

推荐在容器中运行 OpenClaw 以实现最小权限隔离，防止过高系统权限风险。

Terminal
docker pull openclaw/openclaw:latest

docker run -d \
  --name openclaw \
  -p 18789:18789 \
  -v ~/.openclaw:/root/.openclaw \
  -e OPENCLAW_API_KEY=sk-your-key \
  openclaw/openclaw:latest
02

使用 Docker Compose（推荐）

docker-compose.yml
version: '3.8'
services:
  openclaw:
    image: openclaw/openclaw:latest
    container_name: openclaw
    restart: unless-stopped
    ports:
      - "18789:18789"
    volumes:
      - ./openclaw-data:/root/.openclaw
    environment:
      - OPENCLAW_API_KEY=${OPENCLAW_API_KEY}
      - OPENCLAW_MODEL=claude-opus-4
    security_opt:
      - no-new-privileges:true
✅
安全提示：Docker 部署通过容器隔离自动限制了文件系统访问范围， 是企业场景下最推荐的部署方案。
⚡ Skills 技能

可插拔技能生态

一个 Skill 就是一个目录，包含至少一个 SKILL.md，OpenClaw 扫描到即可加载。 通过 ClawHub 生态，安装一行命令即可扩展 AI 能力。

安装技能通用命令 
# 安装技能
clawhub install <skill-name>

# 调用技能
/skill <skill-name> [input]

# 查看已安装技能
openclaw skills list

技能加载路径（优先级从高到低）

最高 <workspace>/skills/ 工作区技能，适合放入代码仓库
中等 ~/.openclaw/skills/ 本地全局技能
默认 bundled skills 随安装包提供的内置技能
💡 使用技巧

从安装到精通

许多用户安装后却不会「真正」用好 OpenClaw。以下三步帮你跨越鸿沟。

👤
第一步 · 最关键

录入个人信息，建立记忆档案

OpenClaw 的持久记忆能力是与普通 AI 工具最大区别。 大多数新手跳过这步，导致体验远不如预期。 你需要主动告诉 OpenClaw 你是谁、你的职业、偏好和工作习惯， 它会将这些信息长期保存，下次对话自动带入。

示例指令 我叫张伟，是一名后端工程师，主要使用 Python 和 Go， 工作时间是工作日 9:00-18:00，请记住这些信息。
🧩
第二步 · 核心配置

优先安装六大核心技能

新手用户建议优先安装以下六个技能，覆盖最常用场景：

✓ CapabilityEvolver — 自适应能力进化
✓ self-improving-agent — AI 自我学习优化
✓ Summarize — URL 和文件内容摘要
✓ Find Skills — 技能生态导航
✓ Agent Browser — 浏览器自动化
✓ 办公套件技能 — Gmail / Notion / 飞书
🔗
第三步 · 进阶提升

多工具协作，构建工作流

OpenClaw 最强大之处在于它作为「协调层」，将多个工具结合使用。 通过「消息即指令」的交互方式，在 Telegram 或飞书中发送自然语言， 即可触发跨工具的复杂工作流。

📥 收到邮件 → 📝 自动摘要 → 📋 写入 Notion → 📨 发送飞书通知

🔒 安全指南

工信部「六要六不要」安全建议

工信部 NVDB 联合多方机构发布针对 OpenClaw 的官方安全建议， 部署前务必了解并落实以下安全规范。

✅

使用官方最新版本

从官方渠道下载最新稳定版本，开启自动更新提醒；升级前备份数据，升级后验证补丁是否生效。

❌ 不要 使用第三方镜像版本或历史版本
🌐

严格控制互联网暴露面

定期自查是否存在公网暴露，确需互联网访问可使用 SSH 等加密通道，并限制访问源地址，使用强密码认证。

❌ 不要 将智能体实例直接暴露到互联网
🔐

坚持最小权限原则

根据业务需要授予最小必需权限，重要操作进行二次确认或人工审批，优先在容器或虚拟机中隔离运行。

❌ 不要 在部署时使用管理员权限账号
🧩

谨慎使用技能市场

审慎下载 ClawHub 技能包，安装前审查技能包代码，建立技能白名单机制，强制进行代码审计与沙箱测试。

❌ 不要 使用要求「下载 ZIP」「执行 shell 脚本」或「输入密码」的技能包
🛡️

防范社会工程学攻击

使用浏览器沙箱功能，防范浏览器劫持攻击；不轻信通过 AI 传递的外部指令，对异常指令保持警惕。

❌ 不要 在内网环境部署未经审计的公共 Skill
📋

建立完整审计机制

加强凭证管理，避免在环境变量中明文存储密钥；建立完整的操作日志审计机制，确保满足合规要求。

❌ 不要 在生产环境直接授予删除文件、发送数据等高危权限
